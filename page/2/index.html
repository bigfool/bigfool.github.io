Ubuntu14.04Drozer工具的安装和使用

Ubuntu14.04 Drozer工具的安装和使用

1Drozer工具简介

略

2工具的安装

2.1安装前提

确保PC机的操作系统里面安装了:

  • JRE或者JDK
  • Android SDK

64位的Ubuntu14.04需要安装IA32-libs。 确保adb和java已经在系统环境变量PATH中。

2.2Ubuntu安装Drozer

可以在官网https://www.mwrinfosecurity.com/products/drozer/community-edition/下载.deb的安装包直接安装;也可以下载.egg的python包安装，此时需要自己手动下载安装protobuf和twisted包。我是直接下载的.deb包安装的。

2.3手机或者模拟器安装代理agent.apk

下载agent.apk，使用命令安装：

adb install agent.apk

3 Drozer的使用

3.1建立Drozer Console和Dorzer Agent的连接，开启一个会话

Step1建立端口转发，Drozer默认使用31415端口。 如下命令：将Ubuntu的TCP端口31415的所有数据转发到手机上31415端口:

adb forward tcp:31415 tcp:31415

Step2在Android设备上开启Agent server。 选择”Enbeded Server“那一项，然后选择”Enadble“开启Agent server。

Step3使用drozer Console连接Agent server。 在Ubuntu的终端中输入命令：

drozer console connect

打开效果如下：

3.2Drozer Console的命令

Drozer Console是一个命令行环境，和bash shell相似。 Drozer Console提供了一系列的评估Android设备安全性的模块，每个模块有很明确的功能。 下图是自己根据官方文档翻译出的命令描述，可能不准确：

命令 描述 备注
run MODULE 运行一个drozer模块
list或者ls 显示在当前会话中可被执行的所有drozer模块 隐藏你没有权限执行的模块
shell 在设备上的Agent进程上下文中开启一个交互式的Linux shell
cd 跳转命令，挂载到指定的模块空间 可以避免重复输入模块的全名
clean 清除drozer保存在Android设备上的临时文件
contributors 列出为drozer框架做出贡献的人名
echo 在console中显示文本
exit 结束drozer会话
help ABOUT 显示某一个命令或者模块（ABOUT）的帮助
load 加载并且顺序运行一个包含一系列drozer命令的文件
module 从互联网上寻找和安装drozer的扩展模块
permissions 显示drozer Agent获得的权限
set 赋值运算，将一个数值存到一个变量中，这个变量可以做环境变量 如：set HOST=193.168.1.1
unset 取消赋值 ……

Android逆向常用工具说明

Android逆向常用工具说明

Android逆向常用的工具是Apktool，dex2jar，jd-gui。其中Apktool是google官方出的反编译工具，用来将整个apk软件包反编译出来相应的资源和smali源码。dex2jar是将apk软件包中的.dex文件转换成.class的java字节码文件，jd-gui是将.class文件反编译成java源码。

本人的安装环境是Kubuntu14.04

1 Apktool的安装

下载两个文件：apktool1.5.2.tar.bz2和apktool-install-linux-r05-ibot.tar.bz2，因为是google官方出品的，下载地址是google code里面，前段时间被封的厉害，费了一番功夫才下载成功，传到百度网盘，方便大家下载：http://pan.baidu.com/s/11tvpc

分别解压缩这两个文件，获得三个文件：apktool.jar和aapt，apktool

可以将这三个文件放到Android SDK目录的tools目录下，前提是这个目录已经添加到环境变量PATH里面了，这样以后就可以直接在终端中里面使用了。

然后在终端中输入apktool，验证是否安装成功：

1 dex2jar和jd-gui的安装

分别下载：dex2jar-0.0.9.15.zip和jd-gui-0.3.5.linux.i686.tar.gz，dex2jar的代码也是托管在google code上，所以在我共享的百度网盘里面也有。Jd-gui的下可以去它的官网：http://jd.benow.ca/，如果使用eclipse的话，还可以下载个jd-gui的eclipse插件。

解压dex2jar-0.0.9.15.zip和jd-gui-0.3.5.linux.i686.tar.gz

dex2jar-0.0.9.15.zip解压出来的文件太多了，于是直接将其文件夹路径添加了环境变量的PATH里面。 jd-gui-0.3.5.linux.i686.tar.gz解压出来的jd-gui文件直接丢到了Android SDK的tools目录里，同apktool一样。

验证一下： apktool没有问题。

但jd-gui提示错误：jd-gui: error while loading shared libraries: libgtk-x11-2.0.so.0: cannot open shared object file: No such file or directory 这是因为我的系统没有libgtk的库吧，解决办法apt-get安装相应的库文件:

sudo apt-get install libgtk2.0-0:i386 libxxf86vm1:i386 libsm6:i386 lib32stdc++6

然后就能正常运行了。

3 安装设置jd-gui的eclipse插件jdeclipse

插件的安装就不赘述了，安装成功后，在菜单栏Windows->Preferences->General->Editors->FileAssociations将.class文件类型关联为Class File Editor。

Dare反编译工具的安装　

Dare反编译工具的安装

最近在看Android安全方面的文献，从一篇论文《A Study of Android Application Security》里面发现了反编译工具ded，作者Wiliam Enk，宾夕法尼亚州立大学的。作者因为研究Android应用的安全而开发了ded，在这篇论文里面还介绍了ded的设计原理。我就想用用，发现作者的ded工具已经不更新了，但是作者对ded的性能表现不是很满意，于是就又开发出了ded工具的升级版Dare反编译工具。本人对比Dare的安装和ded安装，发现Dare安装包里面已经默认包含了soot和Jasmin，这样Dare的安装就简单了许多。 作者还开源了Dare的源码，托管在Github上，地址是：https://github.com/dare-android

我的安装环境是64位的KUbuntu14.04。参考Dare的安装说明：http://siis.cse.psu.edu/dare/installation.html

1 安装JDK

这个就不赘述了。

2 安装依赖库

因为我的Kubuntu是64位，按说明需要安装ia32-libs，但是我的Ubuntu14.04在安装的时候说软件库里面没有ia32-libs，只好安装了它建议的几个软件lib32z1, lib32ncurses5, lib32bz-1.0。

3 下载安装Dare

先下载Dare的安装包：https://github.com/dare-android/platform_dalvik/releases/download/dare-1.1.0/dare-1.1.0-linux.tgz

解压之，并进入dare目录，然后运行命令：./dex-preopt –bootstrap

这个将在预校验（preverification）阶段，优化启动引导类。This will optimize the bootstrap classes to allow for the preverification step。

4 修改环境变量

为了使用方便，将dare目录添加到环境变量PATH中，打开/etc/profile在后面添加：

export PATH=/home/qizhf/soft/installedsoft/dare-1.1.0-linux:$PATH

然后换个目录，输入dare就会出现提示：

这样以后在任何目录下都可以运行dare工具了。

5 深入阅读

Dare作者的这篇文章Retargeting Android Applications to Java Bytecode叙述了Dare的设计原理。

迁移博客到git pages，采用hexo

原来的博客好久没更新，主机提供商换了IP，发了邮件，但是懒得看，才发现原来的网站挂了一个多月。刚开始建站写博客时，找wordpress的各种主题，弄的花里胡哨的。后来发现还是简洁点比较优美，也不想再买空间了，就迁移到github上吧。　

1安装hexo

sudo apt-get install npm　＃会自动安装nodejs

sudo npm install hexo -g

解决/usr/bin/env: node: 没有那个文件或目录

#hexo version
/usr/bin/env: node: 没有那个文件或目录

产生的原因安装了nodejs，hexo是用node，所以查看下nodejs可执行文件在哪个目录下，新建一个软链接：

#whereis nodejs
nodejs: /usr/bin/nodejs /usr/lib/nodejs /usr/bin/X11/nodejs /usr/include/nodejs /usr/share/man/man1/nodejs.1.gz

#sudo ln -s /usr/bin/nodejs /usr/bin/node

2建本地博客

mkdir blog
hexo init kechuandai-hexo
cd kechuandai-hexo
npm install

然后开启hexo server

hexo server

在浏览器中输入：http://127.0.0.1:4000

如果打开则成功。

3本地添加文章

hexo new 文章名
hexo generate
hexo deploy

4.1 新建一个git　pages　repo
