1.2 Android SDK的安装

S1：下载Android SDK。 S2：使用tar zxvf解压sdk的压缩包到/home/qizhf/soft/installedsoft目录下。
S3：读SDK Readme文档知道：这刚解压出来的SDK只包含基本的sdk工具， 如果想开始开发应用程序，必须安装Platform-tools 和至少某一个版本的Android platform。
Platform-tools包含的是经常变化更新的编译工具，比如adb，dexdump等。 在tools目录下运行android，可以打开SDK manager，这是图形界面的。 运行android update sdk –no-ui 非图形界面的。 S4：终端中，在tools目录下输入./android打开SDK manager。 选择要下载的SDK版本，然后下载。

出现了错误，因为GFW的原因吧： 解决android SDK安装无法下载的问题：修改host文件或者使用VPN。 在网上找了个能ping通的dl-ssl.google.com 的IP：203.208.46.146 将其添加到/etc/hosts文件中： 然后我重启了机器，重新运行SDK manager。需要在SDK Manager的菜单里设置：tools->option选择强制强https强求转为http，不这样的话，会发现有证书错误，不能下载。 终于搞定。 下载速度很快啊。 S5: 修改环境变量。 在 /etc/profile中添加如下部分 export PATH=/home/qizhf/soft/installedsoft/android-sdk-linux/platform-tools:$PATH export PATH=/home/qizhf/soft/installedsoft/android-sdk-linux/tools:$PATH 然后运行命令：source /etc/profile命令更新环境变量。 S6：换到主目录下，输入adb version 和emulator -vesion看是否真正安装成功。

1.4 Eclipse安装ADT，CDT

1.4.1 安装ADT 在eclipse的菜单栏里 选择help->install new software 在对话框里输入网址：

http://dl-ssl.google.com/android/eclipse/ 安装ADT 安装完成后，重启eclipse，会让选择SDK所在的位置： 安装成功后在菜单栏的Windows菜单下，能找到Android SDK manager和Android Virtual Device Manager。 在File->New Project下可以发现有Android工程的向导。

1.4.2 安装CDT 在eclipse的菜单栏里 选择help->install new software 在对话框里输入网址：
